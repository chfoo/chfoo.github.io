abstract EnvironmentFlags(Int)

package lightninghx

to Int,

Environment options.

Variables

inline read onlyFixedMap:EnvironmentFlags = 0x01

mmap at a fixed address (experimental).

This is not useful in Haxe.

inline read onlyMapAsync:EnvironmentFlags = 0x100000

Don't wait for fsync when WriteMap is specified.

inline read onlyNoLock:EnvironmentFlags = 0x400000

Don't use locking.

The caller will need to manage concurrency themselves.

inline read onlyNoMemInit:EnvironmentFlags = 0x1000000

Don't clear unused memory to zeros before writing to disk.

This favors performance over protecting data leaks. This has no effect when WriteMap is specified.

inline read onlyNoMetaSync:EnvironmentFlags = 0x40000

Don't call fsync (metadata), but fdatasync, after commit.

inline read onlyNoReadAhead:EnvironmentFlags = 0x800000

Don't read additional data around the current data if supported by OS.

This option disables optimization for sequential data access and may improve performance of random data access.

inline read onlyNoSubDir:EnvironmentFlags = 0x4000

Don't use a directory, but use a filename prefix instead.

inline read onlyNoSync:EnvironmentFlags = 0x10000

Don't call fsync after commit.

inline read onlyNoTLS:EnvironmentFlags = 0x200000

Don't associate reader lock slots to threads.

This option associates readers to transactions instead of threads. Read-only transactions can be shared by threads and multiple read-only transactions can be held by a thread.

inline read onlyReadOnly:EnvironmentFlags = 0x20000

Open as read-only.

inline read onlyWriteMap:EnvironmentFlags = 0x80000

Avoid malloc to get pages and write directly to the memory map.

This favors performance over safety. It is possible for buggy code to overwrite wrong pages and corrupt the database.

Do not mix this option with other processes.
