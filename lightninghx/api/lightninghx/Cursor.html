LightningHx API documentation

interface Cursor

package lightninghx

implemented by CPPCursor, SafetyCursor

A handle to navigate and manipulate items.

A cursor belongs to a transaction and database.

Since a cursor belongs to a transaction, it is not thread safe.

Methods may throw exceptions except for getTransaction() and getDatabase().

close() must be called to release resources held by this instance. A cursor will automatically be closed in a write transaction.

Methods

close ():Void

Release resources held by this instance.

Do not call this method if this cursor belongs to an uncommitted transactions.

This instance must not be used after calling this method except for renew() in a read-only transaction.

count ():Int

Return the number of duplicates for the current key.

This method should only be called for DUPSORT databases.

delete (?flags:Flags<WriteFlags>):Void

Remove key-data pair at the the current position.

Parameters:

flags

Cursor options.

get (operation:CursorOperation, ?key:Bytes, ?data:Bytes):KeyDataPair

Get key-data pair from the database.

The returned key and data is a reference to the memory map and must not be modified.

Parameters:

operation

Where and how to position the cursor.

key

Key to search depending on operation.

data

Data to search depending on operation.

getDatabase ():Database

Return the associated database.

getTransaction ():Transaction

Return the associated transaction.

put (key:Bytes, data:Bytes, ?flags:Flags<WriteFlags>):Void

Store key-data pair at the the current position.

Parameters:

key

Key of the pair.

data

Data of the pair.

flags

Cursor options.

renew (transaction:Transaction):Void

Open the cursor again in a read-only transaction.

Parameters:

transaction

Read-only transaction.
