LightningHx API documentation

interface Environment

package lightninghx

implemented by CPPEnvironment, SafetyEnvironment

A handle to the memory mapped environment.

An environment's methods are thread-safe except for those noted. The methods may throw an Exception except for close().

close() must be called to release resources held by this instance.

Methods

beginTransaction (?flags:Flags<EnvironmentFlags>):Transaction

Returns a new Transaction.

Parameters:

flags

Transaction options.

close ():Void

Release resources held by this instance.

This method is not thread safe. This instance must not be used after calling this method.

copy (path:String, ?flags:Flags<CopyFlags>):Void

Copy the environment to another directory.

Because this operation is a long-lived read-only transaction, the file can grow when there are write transactions.

Parameters:

path

Directory name.

flags

Copy options.

getFlags ():Flags<EnvironmentFlags>

Return the environment options.

getMaxKeySize ():Int

Return the maximum size of the key.

Default is 511 and can be changed by recompiling LMDB.

getMaxReaders ():Int

Return the maximum number of read transactions.

getPath ():String

Return the path given in open().

info ():EnvironmentInfo

Return information about the environment.

open (path:String, ?flags:Flags<EnvironmentFlags>, ?unixMode:Int):Void

Open the environment.

Setting various options, such as the page size, must be done before opening the environment.

Once the environment is opened, it cannot be opened again (from within this process) until it is closed. Share this environment with threads instead.

Parameters:

path

Directory name (or filename if appropriate flags is given).

flags

Options for the memory map.

unixMode

UNIX permissions such as 0o660.

readerCheck ():Int

Clear stale entires in the reader lock table.

Returns the number of stale entries removed.

setFlags (flags:Flags<EnvironmentFlags>, clear:Bool = false):Void

Set additional options to those given in open().

This method is not thread safe.

Parameters:

flags

Environment options.

clear

Whether to remove the options.

setMapSize (size:Int64):Void

Set the maximum size of the database.

This value is 10 MB by default and will throw exceptions if the database grows beyond that. The size should be very large and should be a multiple of the OS page size. This value is persisted after the first write transaction.

For details, please see the C documentation for the mdb_env_set_mapsize() function.

Parameters:

size

Size in bytes.

setMaxDatabases (count:Int):Void

Set the maximum number of named databases.

If other databases besides the unnamed default is required, set this to the number needed. Performance may be low for large numbers.

setMaxReaders (size:Int):Void

Set the maximum number of read transactions.

This method can only be called by the first process to open the environment.

stat ():Statistics

Return statistics about the environment.

sync (force:Bool = false):Void

Flush the data buffers to disk (fsync).

Sync is called automatically during transaction commit unless disabled by option.

Parameters:

force

Whether to block until the OS has finished fsync.
