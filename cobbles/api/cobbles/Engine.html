Cobbles API Documentation

class Engine

package cobbles

implements Disposable

Available on all platforms

Represents layout and render engine.

An engine is used repeatedly to process text.

You can have multiple engines such as one engine for GUI elements that do not change frequently and multiple engines for user chat room messages.

Constructor

new(library:Library)

Variables

font:FontID

The current font face.

Default is 0 (not corresponding to a valid font face).

fontSize:Float

The current font size in points.

Default is 12.

If the font file contains bitmaps and not vectors, the value represented is in pixels.

language:String

The current language as a BCP 47 language tag.

This low-level value typically controls the glyph variant selection to use from a font file.

Default value is an empty string which indicates automatic detection.

A tag is 2 characters or longer.

lineLength:Int

Line length in pixels.

If the value is 0, no word wrapping is performed. Otherwise, lines will be word wrapped at the given pixel width.

locale:String

Locale as a BCP 47 language tag.

This value is used for assisting lower-level functions to tailor the presentation of the text to your application's user. It is typically the GUI's locale or a document's language.

The default is an empty string which indicates automatic detection of the user's locale if possible.

A tag is 2 characters or longer.

@:value(new MarkupParser())markupParser:MarkupParser = new MarkupParser()

An associated markup parser for this engine.

read onlyoutputInfo:OutputInfo

Information about the lay out operation.

script:String

The current script as a ISO 15924 string.

This low-level value typically controls the rules for shaping glyphs.

A ISO 15924 string is a 4 character string such as "Latn".

Default value is an empty string which indicates automatic detection.

scriptDirection:ScriptDirection

The current script direction.

Default value "not specified" indicates automatic detection.

textAlignment:TextAlignment

Controls the alignment of text in each line.

Default is not specified.

clear():Void

Empty the text buffer.

This empties the text buffer so the engine can be used to process anther set of text. The properties are not reset and tiles are not cleared.

clearTiles():Void

Clear associated tiles and glyphs.

The library context caches glyphs until no engine has a reference to them. This function can be called to reduce memory usage or clear a full texture atlas. This is especially important if your text sources are from user generated content.

This function doesn't affect properties or buffered text, and does not affect other engines. If you have associated data structures, such as a texture atlas, remember to clear those too.
