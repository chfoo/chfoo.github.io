free ():Void

Removes the memory allocated at the addressed location.

This should only be called for pointers for which the caller has ownership for previously allocated memory.

Calling this more than once is undefined behavior.

A value converted to the type Int, haxe.io.Int64, Float, or Pointer.

Integer data types that fit within 32 bits will be promoted to Int while wider integers will be promoted to haxe.io.Int64.

getBytes (count:Int):Bytes

Available on cpp, hl

Returns a Haxe Bytes using the pointer's value as the underlying data.

See:

getDataView for the purpose of this method. Compared to getDataView this method may be slightly faster but is not portable between targets.

getDataView (count:Int):DataView

Return a data view using the pointer's value as the underlying data.

The pointer is interpreted as the target's native array and used as the data view's underlying bytes data.

This method is useful for easier and faster access than the arrayGet and arraySet methods.

Care must be ensured that the pointer is not freed or else the data view will access invalid memory locations.

The pointer is not automatically freed when the returned data view is garbage collected.

Parameters:

count

Array length in bytes.
