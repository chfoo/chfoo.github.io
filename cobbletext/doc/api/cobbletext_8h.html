Detailed Description

Cobbletext C API.

Start at cobbletext_library_new(), then cobbletext_engine_new().

Whenever a function is described as "can error", check the value of cobbletext_get_error_code() before continuing. If there is an error, the returned value or the internal state may be invalid.

Pointer parameters are copied by the function unless noted. Pointer return types should not be freed and become invalid on the next function call unless noted.

Whenever a pointer is described "free only using library", use Cobbletext's corresponding delete function for that handle. Do not call free() on it.
